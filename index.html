Frontend Guidelines

Frontend guidelines to make team members write consistent & cohesive code together.

Html

Html Principles

Standalone

HTML itself should work without CSS and JavaScript. The content should be accessible, form should be able to submited.

Disable images, CSS and JavaScript to check if it works.

Semantics

Use semantics elements (article,aside,figure,figcaption,header,footer,main,nav,section) instead of generic elements (div,span).

Inline Styles

Avoid inline styles or event handlers

Indentation

Use 2 spaces for indentation. This is the only way to guarantee code renders the same in any environment.




<ul>
..<li>Item</li>
..<li>Item</li>
</ul>

Attribute Order

"HTML attributes should come in this order for easier reading:"
class
id, name
data-*
src, for, type, href, value
title, alt,
role, aria-*

Classes are used most often, so they come first. Ids are more specific and should be used sparingly, for in-page bookmarks, so they come second.




<a class="link" id="link" data-toggle="modal" href="#">Example link</a>

Classes

+ Always use lower case.

+ The classes used for javaScript must start withjs-.




<a href="" class="js-popup">Log In</a>

Comments

Don't use HTML comments, because:

+ HTML comments should be visible only for developer.

+ HTML comments increase file size.

Use comments in backend/templating language:




<?php
// Some comments
?>
<p>Lorem ipsum dolor sit amet.</p>






{# Some comments #}
<p>Lorem ipsum dolor sit amet.</p>

Css

Css Principles

Syntax

Use 2 spaces for indentation

Keep each selector to a single line when use group selector

One space after selector

One space after:for each declaration




// Bad
.selector{padding-left:20px;}

// Good
.selector { padding-left: 20px; }

Each declaration should appear on its own line

End all declaration with a semi-colon

Quote attribute values in selector,[type="text"]

Avoid specifying units for zero values

Use meaningful class names and IDs

Avoid absolute positioning

Avoid !important

Don't use breaks
just to make space in your layout

Avoid hacks unless there is no other way to achieve what you want

Declaration Order

Related property declaration should be grouped together following the order
Box(display, float, position, left, top, width, height and so on)
Border & Background
Text(font-family, font-size, text-transform, letter-spacing and so on)
Other




.selector {
  // Box
  display: block;
  float: right;
  position: absolute;
  top: 0;
  right: 0;
  z-index: 100;
  width: 100px;
  height: 100px;
  margin-left: 20px;

  // Border & Background
  border: 1px solid #e5e5e5;
  border-radius: 3px;
  background-color: #f5f5f5;

  // Text
  font: normal 13px "Helvetica Neue", sans-serif;
  line-height: 1.5;
  color: #333;
  text-align: center;

  /* Other */
  opacity: 0.8;
}

Media Query Placement

Place media queries close to their relevant rule sets. Don't bundle them all in a separate stylesheet or at the end of the document.




.selector {
  property: value;
}
@media (min-width: 40em) {
  .selector {
    property: new-value;
  }
}

Multi-lines Vs Single-line

If a rule set includes only one declaration, put it in a signle line, otherwise, declarations should be split to separate lines.




// signle line
.selector { font-size: 14px; }

// separated lines
.selector2 {
  position: absolute;
  left: 0;
  max-width: 200px;
}

Selectors

Use classes over element tag

Lowercase only

Hyphen over underscore




// Bad
.goToTop {}
.go_to_top {}

// Good
.go-to-top {}

Keep selectors short (less than 3)

Take out unnecessary selector




// Bad
.nav ul li { list-style: none; }
a[src^="https://"] { color: purple; }

// Good
.nav li { list-style: none; }
[src^="https://"] { color: purple; }

Avoid overriding styles




// Bad
.nav li { padding-left: 10px; }
.nav li:first-child { padding-left: 0; }

// Good
.nav li + li { padding-left: 10px; }

Bem

https://en.bem.info/method/

The idea of BEM (Block, Element, Modifier) is to divide the user interface into independent blocks. This allows reuse of existing code.




<!-- Block: pop -->
<!-- Element: pop__header -->
<!-- Modifier: pop_is-green -->

<div class="pop pop_is-green">
  <header class="pop__header"></header>
  <main class="pop__main">
    <form class="pop__form">
      <input type="email" name="email">
      <button>Submit</button>
    </form>
  </main>
  <footer class="pop__footer"></footer>
</div>

<!-- File structure -->
search-form/                           # Directory of the search-form
    __input/                           # Subdirectory of the search-form__input
        search-form__input.css         # CSS implementation of the search-form__input element
        search-form__input.js          # JavaScript implementation of the search-form__input element

    __button/                          # Subdirectory of the search-form__button element
        search-form__button.css
        search-form__button.js

    _theme/                            # Subdirectory of the search-form_theme modifier
        search-form_theme_islands.css  # CSS implementation of the search-form block
                                       # that has the theme modifier with the value islands
        search-form_theme_lite.css     # CSS implementation of the search-form block
                                       # that has the theme modifier with the value lite

    search-form.css                    # CSS implementation of the search-form block
    search-form.js                     # JavaScript implementation of the search-form block

Solutions

Fontfaceobserver:

https://github.com/bramstein/fontfaceobserver

Usefontfaceobserverto detect if the custom fonts are loaded. If they get loaded, set a boolean flag in the session storage and add a '.fonts-loaded' class to the html element.

If this page get reloaded or sub pages are opened, the class will be added immediately.




<!-- Place in <head> -->
<script>
  // add class .fonts-loaded if the fonts were downloaded
  if (sessionStorage.getItem('fontsLoaded')) {
    document.documentElement.classList.add('fonts-loaded');
  }
</script>

<!-- Place in <body> -->
<script>
  if (!sessionStorage.getItem('fontsLoaded')) {
    new FontFaceObserver("My Custom Font")
        .load()
        .then(function(){
            sessionStorage.setItem('fontsLoaded', true);
            document.documentElement.className += " fonts-loaded";
        }).catch(function () {
            sessionStorage.setItem('fontsLoaded', false);
        });
  }
</script>






.heading { font-family: Helvetica, Arial, sans-serif; }
.fonts-loaded .heading { font-family: 'My Custom Font', sans-serif; }
